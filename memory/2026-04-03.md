### Key Learnings for Next Session

#### Zendesk Browser Workflow
1. Navigate to `https://upkid.zendesk.com/agent/search/1?q={email}&type=user`
2. Wait 2.5s for results to load
3. Take snapshot, click the user cell (ref for Name cell in tabpanel)
4. Wait 2s, then evaluate to get `status` and `userId`
5. **If status is "Not Hiring":** skip, set sheet A=TRUE B=FALSE
6. **If status is "Intro Call - Passed":** already done by Erica, just mark sheet A=TRUE
7. **If status is "-" or "No Show":** proceed with full workflow
8. Click Teacher Status button (snapshot ref), wait 500ms, snapshot again to find dropdown menuitems
9. Click "Intro Call - Passed" menuitem (always ref=e138 in the dropdown, but verify)
10. Set notes via evaluate on contenteditable div (find by label "Notes", filter by `rect.width > 0`)
11. **ALWAYS close all ZD tabs** after each candidate: `document.querySelectorAll('button[aria-label="Close this tab"]').forEach(b => b.click())`
12. If email not found by search, try searching by name
13. Some candidates have different emails in Zendesk vs sheet (e.g., @byu.edu vs @gmail.com)

#### Retool Browser Workflow
1. On Teachers tab, click search textbox (ref=e13), Cmd+A, type App ID, click Search button (ref=e17)
2. Wait 3s for results
3. **CRITICAL:** Always use snapshot ref to click View button — NEVER use `querySelectorAll('button')` because there's a hidden "Honors Academy Incorporated" center View button that intercepts clicks
4. Wait 3s for dialog to load
5. Verify dialog heading shows correct teacher name
6. Verify correct state tab is selected (Utah/Georgia/Arizona)
7. Click Interviewed checkbox (always ref=e25 in the dialog)
8. Press Escape to close dialog
9. If dialog shows a center (like "Honors Academy") instead of teacher, press Escape and use the snapshot ref for the correct View button

#### Sheet Updates
- Use Sheets API for all checkbox updates — never browser
- `A{row}` = Reviewed checkbox (TRUE)
- `A{row}:B{row}` = Reviewed + Erica Approved (TRUE, FALSE for skips)
- Zendesk notes come from column M

#### Edge Cases Encountered
- **"Not Hiring" in Zendesk:** ~50% of early rows. Skip and uncheck Erica Approved.
- **"No Show" in Zendesk:** Override to "Intro Call - Passed" (VideoAsk replaced missed call). Tyler approved this.
- **Email mismatches:** Some candidates registered with different emails in Zendesk vs VideoAsk (e.g., @byu.edu, @icloud.com vs @gmail.com). Search by name as fallback.
- **Dual accounts:** Some candidates have old+new app accounts with different User IDs. Use the sheet's App ID for Retool.
- **Erica processing concurrently:** Some candidates get done by Erica while we're working. Check if already "Intro Call - Passed" before modifying.
- **Existing notes:** When Zendesk already has notes, APPEND new note (don't replace). Use `existing + '\n' + new_note`.
- **Service Map tab hijack:** If Retool accidentally switches to Service Map tab, click "Teachers" tab to get back.
- **Google Sheet Name Box trap:** NEVER use Name Box in browser — it can rename the document. Always use Sheets API.
